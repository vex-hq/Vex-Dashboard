## What's new

Vex now supports **A/B testing for AI agents**. Create experiments with multiple variants, tag your agent executions via the SDK, and compare performance metrics side-by-side in the dashboard.

### Experiments & Variants

Create experiments directly from the dashboard with named variants (e.g. "control" vs "treatment"). Each experiment gets a unique ID that you pass to the SDK when tracing executions.

```python
from vex import Vex, VexConfig

vex = Vex(api_key="your-key", config=VexConfig(mode="sync"))

with vex.trace(
    agent_id="support-bot",
    task="Answer customer questions",
    input_data={"query": user_query},
    experiment_id="47389aeb-67b7-4fa5-aad2-d974ec1a7260",
    variant="treatment",
) as ctx:
    response = my_agent.run(user_query)
    ctx.record({"response": response})
```

### Variant Comparison Dashboard

The experiment detail page shows side-by-side metrics for every variant:

- **Avg Confidence** — overall verification confidence per variant
- **Pass / Flag / Block Rate** — how often each variant passes, gets flagged, or gets blocked
- **Correction Rate** — how often corrections are applied
- **Avg Latency & Cost** — performance characteristics per variant
- **Per-check scores** — hallucination, drift, schema, and coherence scores broken down by variant

The best-performing variant is highlighted with a "Best" badge on each metric.

### Datasets

Upload test datasets with input/expected output pairs for offline evaluation. Use them with experiments to run controlled comparisons against known-good data.

### SDK Support

Both SDKs now include `experiment_id` and `variant` parameters:

- **Python SDK** `vex-sdk==0.4.0` — `vex.trace(..., experiment_id="...", variant="...")`
- **TypeScript SDK** `@vex_dev/sdk@0.2.0` — `vex.trace({ experimentId: "...", variant: "..." })`
